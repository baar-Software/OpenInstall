# Releasing OpenInstall

OpenInstall releases are built on GitHub Actions. Windows code signing through
the SignPath Foundation is planned, but not yet enabled for public releases.

## Prerequisites

Repository configuration:

* Repository Actions permissions allow release creation.

Local maintainer tools:

* Stable Rust.
* Node.js 20 or newer.
* Tauri CLI v2.

## Pre-Release Checklist

1. Update `CHANGELOG.md`.
2. Confirm version numbers in:
   * workspace `Cargo.toml`
   * `src-tauri/Cargo.toml`
   * `src-tauri/tauri.conf.json`
   * `ui/package.json`
3. Run:

   ```sh
   cargo fmt --all --check
   cargo clippy --workspace --all-targets -- -D warnings
   cargo test --workspace
   npm --prefix ui ci
   npm --prefix ui test
   npm --prefix ui run build
   ```

4. Build locally if desired:

   ```sh
   cargo tauri build --bundles msi
   ```

5. Verify docs mention the right package format and safety model.

## Cut A Release

Create and push a signed tag:

```sh
git tag -s v0.1.0 -m "OpenInstall v0.1.0"
git push origin v0.1.0
```

The release workflow will:

1. Build the Tauri MSI on `windows-latest`.
2. Stage the MSI as an unsigned release asset.
3. Create/update the GitHub Release and attach the unsigned MSI.

## After Release

* Download the MSI from the GitHub Release.
* Install OpenInstall on a clean Windows test machine.
* Confirm `openinstall://` links open the app.
* Confirm the release notes generated by GitHub are readable.

## Never Do This

* Do not store code-signing private keys in GitHub secrets.
* Until Windows code signing is enabled, label MSI assets clearly as unsigned.
* Do not disable Windows security, SmartScreen, or Windows origin metadata in release
  automation.
